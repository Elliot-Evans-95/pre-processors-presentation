What is a pre-processor

A computer program or language that modifies data to conform with the requirements of another program or language. So in essence, you can write code how you want and it will be complied down to the language that it needs to be in.

Why pre-processors exist

One advantage of using a pre-processor is that you can write code has not yet made it into that particular languages spec. You can write cleaner code or use a way of writing that cannot be yet used and have it complied down to that language. Another advantage of a pre-processor is that it can change the paradigm of how you would write the code for that language. This can be a benefit if want a focus of one way of programming.

Before I move onto to pre-processors I would like to talk about the differences between Compile and transpile as, at times, they can be seen as the same. A compiler usually produces a directly usable artifact. Such as C# and C. So the compiler transforms it into Intermediate language (IL) A transpiler produces another form of source code such as coffeescript (shrug) and babel (ESNext). So the compiler transforms it into javascript

Future of pre processors

From the examples that I have covered, pre-processors are a great aid to developers as they enhance what we can write today but what is there future? I believe that in time, they will be less need for pre-processors to a point there are are not needed at all. There is already work in progress in the Javascript and CSS community to take some parts of pro-processor languages and add them into the language (CSS variables, static methods and classes).
